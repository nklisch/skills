Shared shell — card, filters, auto-hide

The substrate board reads the same .work/ files the agent does, then lays them out for a human: a kanban by stage, a dependency view that makes blockers obvious, and a dense sortable table. Released and archived items hide by default so active work stays in focus.

It is read-only — a window onto the work, not a second place to edit it. The shell owns the item-card and the global filter state; each view just decides how to arrange what the filter lets through. depends_on: [epic-substrate-board-host]

Squint check: the body block should read as an even medium grey at arm's length — Geist at 16/1.5 holds an even typographic color.
