# Briefs

Domain briefs and campaign output live here.

- **Single-topic briefs** — produced by `/research` and `/brief` (one markdown file per topic)
- **Campaign directories** — produced by `/deep-research` (`<campaign-slug>/` with `parent.md` + specialist briefs + quality report)

A brief gates a roadmap phase when its frontmatter declares `blocks_phase: "<phase-id>"`. See the `research-pipeline:build-process` skill for when to write one.
