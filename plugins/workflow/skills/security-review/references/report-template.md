# Security Review Report Template

Use this template when writing the final report. Replace placeholders with actual data.

---

```markdown
# Security Review Report

**Project:** {project name}
**Date:** {date}
**Scope:** {full repo | scoped directory}
**Stack:** {detected languages, frameworks, infrastructure}

## Executive Summary

**Overall Score: {X.X}/10**

{2-3 sentence summary of the security posture. Highlight the most critical finding
and the strongest area.}

### Findings Summary

| Severity | Count |
|----------|-------|
| Critical | {n}   |
| High     | {n}   |
| Medium   | {n}   |
| Low      | {n}   |
| Info     | {n}   |

## Domain Scores

| Domain | Score | Key Issue |
|--------|-------|-----------|
| {domain name} | {X}/10 | {one-line summary of top finding or "No issues found"} |
| ... | ... | ... |

## Findings

### {Domain Name} — {X}/10

#### [{severity}] {Finding title}

**Location:** `{file}:{line}`
**Evidence:**
\`\`\`{lang}
{1-5 lines of problematic code}
\`\`\`
**Impact:** {What could go wrong — be specific about the attack vector}
**Remediation:** {What should change — direction, not implementation}

{Repeat for each finding in this domain, ordered by severity}

---

{Repeat domain sections for each audited domain}

## Positive Findings

{List security measures that are done well — give credit where due. This helps
calibrate the report and avoids a purely negative tone.}

- {Good practice found} (`{file}:{line}`)
- ...

## Recommended Next Steps

1. **Immediate** — {address Critical/High findings}
2. **Short-term** — {address Medium findings, improve coverage}
3. **Long-term** — {hardening measures, process improvements}

---
*Generated by security-review skill*
```
