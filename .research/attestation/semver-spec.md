# Semantic Versioning 2.0.0 — specification

## Summary

The Semantic Versioning 2.0.0 specification, in its numbered clauses, sets out how a
`MAJOR.MINOR.PATCH` version increments. Two clauses bear on the pre-1.0 phase: item 4
addresses major-version-zero (`0.y.z`) and item 5 addresses the `1.0.0` release. Item 4
states that `0.y.z` is for initial development and carries no stability expectation; item 5
states that `1.0.0` is the point at which the public API is defined.

## Key passages

> Major version zero (0.y.z) is for initial development. Anything MAY change at any time. The public API SHOULD NOT be considered stable. — spec item 4

> Version 1.0.0 defines the public API. The way in which the version number is incremented after this release is dependent on this public API and how it changes. — spec item 5
