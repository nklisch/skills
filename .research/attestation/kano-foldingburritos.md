# The Complete Guide to the Kano Model (Folding Burritos)

## Summary

A comprehensive practitioner guide to the Kano model, covering the survey methodology (functional/dysfunctional question pairs), the five category types, how continuous analysis calculates satisfaction coefficients, and methodological limitations.

## Key passages

**Survey methodology:** Each feature receives two questions:
- Functional form: "How do you feel if you *have* this feature?"
- Dysfunctional form: "How do you feel if you *don't have* this feature?"
Both use five response options: "I like it / I expect it / I am neutral / I can tolerate it / I dislike it."

**Evaluation matrix:** The 25 possible response pair combinations map to a standard classification table assigning: Performance, Must-be, Attractive, Indifferent, or Questionable/Reverse.

**Five categories:**
1. **Performance (One-dimensional):** Linear relationship; more implementation = more satisfaction.
2. **Must-be (Basic):** Prevent dissatisfaction but don't create satisfaction when present. Absence causes strong dissatisfaction.
3. **Attractive (Excitement):** Delight through unexpected benefits; absence causes no dissatisfaction.
4. **Indifferent:** No meaningful satisfaction response either way; money sink.
5. **Reverse:** Customer prefers the feature's *absence* — indicates product misalignment.

**Continuous analysis:** Calculates functional and dysfunctional satisfaction coefficients, plots features on a two-dimensional plane with importance weighting; allows comparison across features positioned in similar zones.

**Prioritization heuristic:** Must-be features first, then Performance features, then a selection of Attractive features.

**Methodological limitations explicitly stated:**
- Results are a temporal snapshot; Attractive features naturally decay into Must-be categories over time (the "Kano decay" effect).
- Discrete analysis collapses 25 response combinations into one category, losing granular information.
- Importance self-reporting may not reflect actual purchasing behavior.
- Customer segmentation is critical — homogeneous analysis obscures persona-specific preferences.
- Automation challenge explicitly named: "Converting responses to categories is algorithmic, but the prioritization decisions between closely-positioned features demand human judgment and business context that resist full automation."

## Structural notes

- The survey-to-category conversion is mechanical/algorithmic; the subsequent prioritization judgment is not.
- Kano does not produce a sortable numeric score for individual items — it produces a category assignment.
- Category assignment is contingent on survey administration (human ceremony to gather data) and decays over time.
- Kano is designed for feature-level product strategy decisions, not for sprint-level queue ordering.
