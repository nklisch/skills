# gate-patterns writes the `.agents/rules/patterns.md` digest

## Brief

Update the `gate-patterns` skill so that, in addition to writing detailed pattern
files to `.agents/skills/patterns/` (unchanged single source of truth), it
generates a concise `.agents/rules/patterns.md` digest that the hook loader
injects. The digest is the slug + one-liner index plus a pointer line telling the
agent to load the `patterns` skill / read `.agents/skills/patterns/<slug>.md` for
full detail, examples, and violations. Repoint gate-patterns' "rules folder
directive" at the generic `.agents/rules/`.

Per the Codex review, the digest carries a generated banner ("generated by
gate-patterns — do not hand-edit") and a hash of the source skill index so drift
is detectable; `.agents/skills/patterns/` remains the only hand-maintained source.
Digest and detail are produced in one gate-patterns pass, so they cannot drift.

Does NOT cover: changing the patterns skill format, or the consumers of
`.agents/skills/patterns/` (gate-cruft, gate-docs, refactor-design,
implement-orchestrator) — they are unchanged because the skill stays put.
