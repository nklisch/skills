# gate-patterns writes the `.agents/rules/patterns.md` digest

## Brief

Update the `gate-patterns` skill so that, in addition to writing detailed pattern
files to `.agents/skills/patterns/` (unchanged single source of truth), it
generates a concise `.agents/rules/patterns.md` digest that the hook loader
injects. The digest is the slug + one-liner index plus a pointer line telling the
agent to load the `patterns` skill / read `.agents/skills/patterns/<slug>.md` for
full detail, examples, and violations. Repoint gate-patterns' "rules folder
directive" at the generic `.agents/rules/`.

Per the Codex review, the digest carries a generated banner ("generated by
gate-patterns — do not hand-edit") and a hash of the source skill index so drift
is detectable; `.agents/skills/patterns/` remains the only hand-maintained source.
Digest and detail are produced in one gate-patterns pass, so they cannot drift.

Does NOT cover: changing the patterns skill format, or the consumers of
`.agents/skills/patterns/` (gate-cruft, gate-docs, refactor-design,
implement-orchestrator) — they are unchanged because the skill stays put.

## Architectural choice

The patterns SKILL.md index stays the single source. In the SAME gate-patterns
pass that regenerates `.agents/skills/patterns/SKILL.md` (Phase 5), also emit a
generated digest `.agents/rules/patterns.md` for the hook to force-load. Digest =
a "do-not-hand-edit" banner + a `src-sha256` of the index entries (drift
detectable) + the slug+one-liner entries + a pointer line telling the agent to
load the `patterns` skill / read `.agents/skills/patterns/<slug>.md` for full
detail, examples, and violations. Generated together → cannot drift.

## Implementation Units

Edits to `plugins/agile-workflow/skills/gate-patterns/SKILL.md`. No child stories.

### Unit 1: Emit `.agents/rules/patterns.md` in Phase 5
After regenerating the skill index, write `.agents/rules/patterns.md`:
```markdown
<!-- generated by agile-workflow gate-patterns — do not hand-edit -->
<!-- agile-workflow:patterns-digest src-sha256=<digest of the index entries> -->
## Project Code Patterns
- <slug> — <one-line rule>
- ...

Load the `patterns` skill (read `.agents/skills/patterns/<slug>.md`) for full
rationale, examples, and common violations.
```
The one-liners are the same entries written to the skill index — compute once,
write both.

### Unit 2: Repoint the Phase 1 "rules folder directive"
Phase 1's directive (and the skill description) now says the hook-loaded rules
digest lives in `.agents/rules/patterns.md` (generated), while
`.agents/skills/patterns/` remains the detailed source of truth. Do not treat
`.agents/rules/patterns.md` as hand-editable.

### Unit 3: Commit + mirror notes
Phase 8 commit `git add` includes `.agents/rules/patterns.md`. Phase 6 Claude
mirror note: `.agents/rules/patterns.md` is generated (not a `.claude` mirror
target); only `.agents/skills/patterns/` mirrors.
